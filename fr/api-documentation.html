Authentification

Toutes les requêtes API doivent inclure un en-tête d'autorisation . Cet en-tête doit faire référence à votre clé privée.

Lorsque vous souscrivez à l'accès API, Unlimited leads attribue une clé privée API à votre compte. Votre clé privée API est uniquement connue de vous et d'Unlimited leads. Il est important de préserver la confidentialité de ces deux éléments pour protéger votre compte. Votre clé API est accessible depuis le lien « Compte ».

API Appel : trouver l'e-mail

POST https://api.unlimited-leads-to.online

Renvoie un objet JSON contenant l'adresse e-mail de la personne et du domaine indiqués. Les appels à API sont mesurés et facturés si une adresse e-mail est renvoyée avec le résultat. En l'absence d'adresse e-mail, aucun frais ne sera facturé.

Paramètres de la requête POST

  • firstName - Le prénom de la personne (obligatoire)
  • lastName - Le nom de famille de la personne (obligatoire)
  • domaine - Le domaine à interroger (obligatoire)

Exemples de paramètres de requête

  • prénom - John
  • nom de famille - Doe
  • domaine - company.com

Réponse POST

{ "email": "john.doe@company.com" }

Exemple de requête curl

> curl -vk \ -H "Content-Type: application/json" \ -H "Authorization: api_private_key" \ -X POST "https://api.unlimited-leads-to.online" \ -d '{ "firstName": "John", "lastName": "Doe", "domain": "company.com", }'

Aucune réponse par e-mail trouvée

{ "email": null }
