API Unlimited leads

Authentification

Toutes les requêtes API doivent inclure un en-tête d'autorisation . Cet en-tête doit faire référence à votre clé privée.

Lorsque vous vous inscrivez pour accéder à l'API, Unlimited leads attribue une clé API privée à votre compte. Votre clé API privée est connue uniquement de vous et de Unlimited leads. Il est important de préserver la confidentialité de ces deux informations pour protéger votre compte. Votre clé API est accessible depuis le lien « Compte ».

Appel d'API : rechercher un e-mail

PUBLICATION https://api.unlimited-leads-to.online

Renvoie un objet JSON contenant l'adresse e-mail de la personne et du domaine indiqués. Les appels à l'API sont mesurés et facturés si une adresse e-mail est renvoyée avec le résultat. En l'absence d'adresse e-mail, aucun frais ne sera facturé.

Paramètres de la requête POST

  • firstName - Le prénom de la personne (obligatoire)
  • lastName - Le nom de famille de la personne (obligatoire)
  • domaine - Le domaine à interroger (obligatoire)

Exemples de paramètres de requête

  • prénom - John
  • nom de famille - Doe
  • domaine - company.com

Réponse POST

{ "email": "john.doe@company.com" }

Exemple de requête curl

> curl -vk \ -H "Type de contenu : application/json" \ -H "Autorisation : api_private_key" \ -X POST "https://api.unlimited-leads-to.online" \ -d '{ "firstName": "John", "lastName": "Doe", "domain": "company.com", }'

Aucune réponse par e-mail trouvée

{ "email": null }
