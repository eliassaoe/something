API Unlimited leads

Autenticação

Todas as solicitações de API devem incluir um cabeçalho de autorização . O cabeçalho deve se referir à sua chave privada.

Ao se cadastrar para acesso à API, Unlimited leads atribui uma chave privada de API à sua conta. Sua chave privada de API é de conhecimento exclusivo de você e Unlimited leads. É importante manter a confidencialidade de ambas para proteger sua conta. Sua chave de API pode ser acessada no link "Conta".

Chamada de API: encontrar e-mail

POSTAGEM https://api.unlimited-leads-to.online

Retorna um objeto codificado em JSON com o endereço de e-mail da pessoa e do domínio fornecidos. As chamadas para a API são medidas e cobradas se houver um e-mail retornado com o resultado. Se não houver um e-mail disponível, você não será cobrado.

Parâmetros de solicitação POST

  • firstName - O primeiro nome da pessoa (obrigatório)
  • lastName - O sobrenome da pessoa (obrigatório)
  • domínio - O domínio a ser consultado (obrigatório)

Parâmetros de solicitação de exemplo

  • primeiroNome - John
  • sobrenome - Doe
  • domínio - company.com

Resposta POST

{ "e-mail": "john.doe@company.com" }

Exemplo de solicitação curl

> curl -vk \ -H "Tipo de conteúdo: application/json" \ -H "Autorização: api_private_key" \ -X POST "https://api.unlimited-leads-to.online" \ -d '{ "nome": "John", "sobrenome": "Doe", "domínio": "company.com", }'

Nenhuma resposta de e-mail encontrada

{ "e-mail": nulo }
