Unbegrenzt leads API

Authentifizierung

Alle API-Anfragen müssen einen Autorisierungsheader enthalten. Der Header sollte auf Ihren privaten Schlüssel verweisen.

Wenn Sie sich für den API-Zugriff anmelden, weist Unlimited leads Ihrem Konto einen privaten API-Schlüssel zu. Ihr privater API-Schlüssel ist nur Ihnen und Unlimited leads bekannt. Es ist wichtig, beide vertraulich zu behandeln, um Ihr Konto zu schützen. Ihr API-Schlüssel ist über den Konto-Link zugänglich.

API Anruf: E-Mail finden

POST https://api.unlimited-leads-to.online

Gibt ein JSON-kodiertes Objekt mit der E-Mail-Adresse der angegebenen Person und Domäne zurück. Anrufe an API werden gemessen und abgerechnet, wenn eine E-Mail mit dem Ergebnis zurückgegeben wird. Wenn keine E-Mail-Adresse verfügbar ist, erfolgt keine Abrechnung.

POST-Anforderungsparameter

  • firstName – Der Vorname der Person (erforderlich)
  • lastName – Der Nachname der Person (erforderlich)
  • Domäne – Die abzufragende Domäne (erforderlich)

Beispielanforderungsparameter

  • Vorname - John
  • Nachname - Doe
  • Domäne - company.com

POST-Antwort

{ "email": "john.doe@company.com" }

Beispiel für eine curl-Anforderung

> curl -vk \ -H "Inhaltstyp: application/json" \ -H "Autorisierung: api_private_key" \ -X POST "https://api.unlimited-leads-to.online" \ -d '{ "Vorname": "John", "Nachname": "Doe", "Domäne": "company.com", }'

Keine E-Mail-Antwort gefunden

{ "email": null }
