API __UNLIMITED_LEADS__

Autenticación

Todas las solicitudes de API deben incluir un encabezado de autorización . Este encabezado debe hacer referencia a su clave privada.

Al registrarte para acceder a la API, __UNLIMITED_LEADS__ asigna una clave API privada a tu cuenta. Tu clave API privada solo la conocen tú y __UNLIMITED_LEADS__. Es importante mantener la confidencialidad de ambas para proteger tu cuenta. Puedes acceder a tu clave API desde el enlace de la cuenta.

Llamada API: buscar correo electrónico

PUBLICACIÓN https://api.unlimited-leads-to.online

Devuelve un objeto codificado en JSON con la dirección de correo electrónico de la persona y el dominio especificados. Las llamadas a la API se miden y facturan si se devuelve un correo electrónico con el resultado. Si no hay correo electrónico disponible, no se facturará.

Parámetros de la solicitud POST

  • firstName - El nombre de la persona (obligatorio)
  • lastName - El apellido de la persona (obligatorio)
  • dominio - El dominio a consultar (obligatorio)

Parámetros de solicitud de ejemplo

  • Nombre - John
  • Apellido - Doe
  • dominio - empresa.com

Respuesta POST

{ "correo electrónico": "john.doe@company.com" }

Ejemplo de solicitud curl

> curl -vk \ -H "Tipo de contenido: aplicación/json" \ -H "Autorización: api_private_key" \ -X POST "https://api.unlimited-leads-to.online" \ -d '{ "nombre": "John", "apellido": "Doe", "dominio": "company.com", }'
