Ilimitado leads API

Autenticación

Todas las solicitudes API deben incluir un encabezado de autorización . Este encabezado debe hacer referencia a su clave privada.

Al registrarse para acceder a API, Unlimited leads asigna una clave privada API a su cuenta. Su clave privada API solo la conocen usted y Unlimited leads. Es importante mantener la confidencialidad de ambas para proteger su cuenta. Puede acceder a su clave API desde el enlace de la cuenta.

API Llamada: buscar correo electrónico

PUBLICAR https://api.unlimited-leads-to.online

Devuelve un objeto codificado en JSON con la dirección de correo electrónico de la persona y el dominio especificados. Las llamadas a API se miden y facturan si se devuelve un correo electrónico con el resultado. Si no hay correo electrónico disponible, no se facturará.

Parámetros de la solicitud POST

  • firstName - El nombre de la persona (obligatorio)
  • lastName - El apellido de la persona (obligatorio)
  • dominio - El dominio a consultar (obligatorio)

Parámetros de solicitud de ejemplo

  • Nombre - John
  • Apellido - Doe
  • dominio - empresa.com

Respuesta POST

{ "correo electrónico": "john.doe@company.com" }

Ejemplo de solicitud curl

> curl -vk \ -H "Tipo de contenido: aplicación/json" \ -H "Autorización: api_private_key" \ -X POST "https://api.unlimited-leads-to.online" \ -d '{ "nombre": "Juan", "apellido": "Doe", "dominio": "empresa.com", }'

No se encontró ninguna respuesta de correo electrónico

{ "correo electrónico": nulo }
