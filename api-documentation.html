Unlimited leads API Documentation

Authentication

All API requests must include an Authorization header with your API key:

Authorization: Bearer YOUR_API_KEY

Your API key is available in your account dashboard.

Rate Limits:
• Email Finder: 1 request every 15 seconds, 1000 per day
• Email Verifier: 1 request every 5 seconds, 1000 per day
• LinkedIn Scraper: 1 request every 10 seconds, 500 per day

1. Find Email

POST https://unlimited-finder-api.hamoureliasse.workers.dev

Find email addresses for any person at any company.

Request Parameters

  • firstName - First name (required)
  • lastName - Last name (required)
  • domain - Company domain like company.com (required)

Example Request

curl -X POST https://unlimited-finder-api.hamoureliasse.workers.dev \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "firstName": "John", "lastName": "Doe", "domain": "company.com" }'

Response - Email Found

{ "email": "john.doe@company.com" }

Response - No Email Found

{ "email": null }

2. Verify Email

POST https://api-unlimited-email-verifier.hamoureliasse.workers.dev

Verify if an email address is valid and can receive messages.

Request Parameters

  • email - Email address to verify (required)

Example Request

curl -X POST https://api-unlimited-email-verifier.hamoureliasse.workers.dev \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "email": "john.doe@company.com" }'

Response - Valid Email

{ "status": "valid", "message": "Accepted", "email": "john.doe@company.com", "domain": "company.com", "deliverable": true, "mx": "mail.company.com" }

Response - Invalid Email

{ "status": "invalid", "message": "Rejected", "email": "fake@company.com", "domain": "company.com", "deliverable": false, "mx": "mail.company.com" }

Response Fields

  • status - "valid" or "invalid"
  • message - "Accepted" or "Rejected"
  • deliverable - true if email can receive messages
  • mx - Mail server for the domain

3. LinkedIn Profile Scraper

POST https://api-unlimited-linkedin-scraper.hamoureliasse.workers.dev/

Extract complete profile information from LinkedIn profiles using the profile URL.

Request Parameters

  • url - LinkedIn profile URL with readable username (required)
Supported URL Format: Only URLs with readable usernames are supported.
✅ https://www.linkedin.com/in/john-doe-123/
❌ https://www.linkedin.com/in/ACoAABXYZ123/ (encrypted IDs not supported)

Example Request

curl -X POST https://api-unlimited-linkedin-scraper.hamoureliasse.workers.dev/ \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "url": "https://www.linkedin.com/in/john-doe-123/" }'

Response - Profile Found

{ "name": "John Doe", "jobTitle": "Software Engineer", "company": "Tech Company Inc", "location": "San Francisco, CA", "industry": "Technology", "website": "techcompany.com", "email": "john.doe@techcompany.com", "education": "Stanford University", "headline": "Software Engineer at Tech Company Inc", "linkedinUrl": "https://www.linkedin.com/in/john-doe-123/", "verified": "true" }

Response - Profile Not Found/Error

{ "error": "Profile not accessible or invalid URL", "name": null, "verified": "false" }

Response Fields

  • name - Full name of the person
  • jobTitle - Current job title
  • company - Current company name
  • location - Location information
  • industry - Industry sector
  • website - Company website
  • email - Email address (if found)
  • education - Educational background
  • headline - LinkedIn headline
  • linkedinUrl - Original LinkedIn URL
  • verified - "true" if profile data was successfully extracted
